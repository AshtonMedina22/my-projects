Initializing OpenAI client
First, we will initialize the OpenAI client and define a function get_completion that sends a system prompt and user prompt to the OpenAI API to generate a completion using the specified model. The system prompt specifies the LLM’s role while answering the user prompt. Since we would essentially like it to be an assistant that retrieves information from a textbook, we will set the system prompt to be something like the following statement:

You're a helpful assistant who looks up answers for a user in a textbook and returns the answer to the user's question. If the answer is not in the textbook, you say, 'I'm sorry, I don't have access to that information.'

We can still improve and modify things; one obvious way is to include citations in the answer result. Even though we included document metadata in our search_text, our prompt did not specify how we wanted to use the metadata. One way to fix this is to add context to our prompt instructions before calling the OpenAI model.

Here are some tips:

We started with the instructions “Your task is to answer the following user question using the supplied search results.“ We should append to this prompt to include how we saved the metadata information into our search text. This could include a description that might look like: “At the end of each search result will be Metadata.” Lastly, we should specify how we want documents cited in the answers. This could include specific metadata elements or styles, such as “Cite the passages, their chunk index, and their URL in your answer.”
